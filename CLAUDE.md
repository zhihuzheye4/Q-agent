# gagent 项目 CLAUDE.md

## 项目概况

- **暂用名**：gagent
- **语言**：Python
- **目标**：多技能编排型 Agent 框架（集合 OpenClaw/Hermes/Codex/Claude 之长）
- **架构**：五层（对话/交互 → 编排/规划 → 记忆 → 技能/工具 → 执行/沙箱）

## 已确立的决策

| 决策项 | 结论 |
|--------|------|
| 开发语言 | Python |
| LLM 接入 | 多模型切换（Claude/OpenAI/Gemini/本地） |
| 优先目标 | 快速原型，先跑起来 |
| 架构原则 | 不用网关、不过度抽象、不搞插件市场 |
| 技能注册 | @skill 装饰器，函数签名即协议 |
| 终端安全 | 三级权限（只读/安全写/危险操作） |
| 子 Agent | 与主 Agent 同时唤醒 |
| 记忆系统 | 专用子 Agent 单独负责 |

## 行为规则

1. **记录原文，禁止加工** — `用户发言记录.md` 一字不改、不拆分、不重组。混乱是数据。
2. **状态与变更分离** — `功能索引.md` 只反映当前存活的功能。删除/修改/重构历史写在 `功能变更记录.md`。
3. **发言与总结 1:1** — `用户发言记录.md` 每新增一条，`核心思想总结.md` 同步新增对应条目。
4. **总结后必须执行** — 提取核心逻辑后转化为行动并执行。禁止在总结后附加评价性语句。
5. **AI 输出纳入监控** — AI 出现逻辑混乱、前后矛盾、擅自偏离指令时，在 `对话记录.md` 中如实标注。
6. **对话记录粒度** — 每次对话一条摘要，不因内容长而拆分。
7. **新窗口启动规则** — 启动时只读本文件（CLAUDE.md）。其余记忆文件只在按规则写入时访问。
8. **工作结束更新规则** — 每次工作完成后更新本文件的"当前阶段""下一步"和"最近对话"部分。
9. **空"下一步"≠项目完成** — "下一步"为空时，AI 必须基于项目概况和架构目标主动生成 3-5 条建议方向。
10. **项目根目录保护** — 任何 shell/文件操作不得直接以项目根目录 `G:/agent` 为工作目录或删除目标；写操作必须先 dry-run 并报告影响范围。

## 记忆文件说明

| 文件 | 用途 | 启动时读取 |
|------|------|-----------|
| `CLAUDE.md`（本文件） | 项目入口 + 行为规则 | ✅ 是 |

| `项目开发地图/功能_XXX.md` | 单个功能详情 | ❌ 否 |
| `项目开发地图/功能变更记录.md` | 功能删除/修改历史 | ❌ 否 |
| `记忆文件夹/对话记录.md` | 每次对话摘要 | ❌ 否 |
| `记忆文件夹/用户发言记录.md` | 用户原始发言（只写） | ❌ 否 |
| `记忆文件夹/核心思想总结.md` | 与发言 1:1 对应（只写） | ❌ 否 |
