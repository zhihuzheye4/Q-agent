# 记忆系统规则

本项目专属记忆体系，所有记忆文件遵循三种权限等级。

## 一、权限三原则

| 等级 | AI 写 | AI 读 | AI 改/删 | 用途 |
|------|------|------|---------|------|
| **L1 可写不可读** | 只追加（append-only） | ✗ | ✗ | 历史事实档案，AI 当记录员 |
| **L2 可写可删改不可读** | ✓ | ✗ | ✓ | 人类维护档案，AI 不读 |
| **L3 完全权限** | ✓ | ✓ | ✓ | AI 工作记忆 |

**核心原则**：越重要的记忆越该归 L1。原因：Claude Code 的 Edit 工具有误删风险，L1 纯追加无 Edit 操作，最安全。

## 二、记忆文件清单

## 三、关键操作规则

### 启动流程（新窗口打开时）
1. 只读两文件：`G:\agent\CLAUDE.md` + `G:\agent\memory\临时记忆.md`
2. 其余 L1/L2 文件**禁止读取**
3. 读完后第一动作：向用户复述临时记忆中的"下一步"，确认后开工

### 对话记录写入
- 每轮对话实时 append 到 `memory/对话记录.md`
- 只记录用户和 AI 的文本，**不记录工具调用指令**
- 单文件超过 500KB 或 200 轮 → 切片为 `对话记录_YYYY-MM_partNN.md` 封存到 `memory/归档/`

### 提问状态触发词
- 开始：用户输入 `现在开始提问` 或 `【提问】`
- 结束：用户输入 `现在结束提问` 或 `【结束提问】`
- 区间内文本逐字追加到 `memory/提问回答记录.md`

### 核心思想提取
- 会话末（用户说"结束/再见/关闭窗口"等）批量生成
- 写入 `memory/核心思想提取.md`，两部分：
  1. 核心思想 / 脑洞 / 项目思路
  2. AI 逻辑分析
- 新会话新节追加，不改旧内容

### 临时记忆覆盖（强制流程）
**临时记忆覆写前必须执行**：
1. 将当前临时记忆完整内容追加到 `memory/编年史记忆.md`（L1）
2. 然后才允许覆写 `memory/临时记忆.md`
3. 编年史每节标注时间戳

### 思维导图更新流程
**AI 禁止直接 Edit `memory/思维导图.md`**。操作流程：
1. 向 `memory/思维导图数据.jsonl` 追加一行事件
   - 新增：`{"action":"add","time":"YYYY-MM-DD","name":"功能名","tech":"技术路线","idea":"技术思路"}`
   - 删除：`{"action":"del","time":"YYYY-MM-DD","name":"功能名","reason":"删除原因"}`
2. 调用脚本：`python scripts/update_mindmap.py`
3. 脚本读取 jsonl → 重新渲染 md（双线制：主线存活功能 + 支线已删除功能）

### ADR 决策记录
- 每个重大架构决策追加一条 ADR-NNN
- 新决策取代旧决策时，新条目标注"取代 ADR-XXX"，**旧条目保留不变**
- 纯 append-only

## 四、行为规则

1. **理解核心思想而非字面执行**：用户表述可能用词不当或不准确，必须提取核心思想后按理解执行。此规则优先级最高。
2. **L1 文件禁止 Edit**：只能用 append（追加）方式写入，禁止使用 Edit 工具替换内容。这是防止误删的根本防线。
3. **L3 文件编辑前先 commit**：临时记忆/思维导图/技能注册/错误追踪这些 L3 文件被 Edit 前应先 git commit，靠历史兜底。
4. **临时记忆覆盖前必走编年史**：违反此规则 = 永久丢失进度快照。
5. **思维导图永不直接 Edit**：必须走 jsonl + 脚本路径。

## 五、文件膨胀控制

- 对话记录：> 500KB 或 > 200 轮 → 切片封存到 `memory/归档/`
- 思维导图 jsonl：永远追加，永不切片（单行事件体量小）
- 编年史：永远追加，永不切片（永久档案）
- 其余 L1：永远追加，永不切片
