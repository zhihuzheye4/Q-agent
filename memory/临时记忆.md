### 关键技术坑（M5/M6 新增）

- **pytest-qt 未装**：项目没装 pytest-qt，UI 测试用 `qapp` fixture（scope="module"，QApplication.instance() or QApplication([])）替代 qtbot
- **QT_QPA_PLATFORM=minimal**：UI 测试需在 import PySide6 前设置 os.environ.setdefault("QT_QPA_PLATFORM", "minimal")，headless 跑通
- **mypy strict parent 参数**：QDialog/QMainWindow 的 parent 必须显式注解 `parent: QWidget | None = None`，否则 no-untyped-def
- **mypy QEasingCurve.OutCubic**：PySide6 stubs 不识别该属性，加 `# type: ignore[attr-defined]`
- **mypy _store: SessionStore | None**：成员变量先 None 后赋值 SessionStore，必须显式类型注解 + TYPE_CHECKING import 避免循环依赖
- **chat_page ToolBubble 挂载位置冲突**：原方案 messages_layout.addWidget 与既有 _clear_messages 的 `while count > 1` 逻辑冲突（会把 tool_bubble 留下、stretch 被清）。改挂到外层 layout（scroll_area 后、input_bar 前），不动 messages_layout，零侵入既有测试
- **mypy _tool_history_window = None 后赋值**：必须在 __init__ 显式声明 `self._tool_history_window: ToolHistoryWindow | None = None`，不能在菜单构造里 ad-hoc 赋值
- **ruff organize imports**：tool_history_window.py 的 from pathlib import Path 在函数内被 ruff 视为顺序错误，提到模块顶部解决
