# Q-agent 临时记忆

更新时间：2026-06-20 11:00

## 一、项目详情

- 项目名：Q-agent
- 路径：G:\agent
- 仓库：https://github.com/zhihuzheye4/Q-agent
- 语言：Python 3.10+
- 定位：类似 Claude Code 的桌面端 AI 工具，纯软件形式对接本地 LLM
- 五层结构：对话/交互 → 编排/规划 → 记忆 → 技能/工具 → 工具调用层
- 关键约束：永不沙箱（仅基本输入校验）、本地 LLM 优先、零第三方依赖起步、每里程碑打 .exe、分发零外部依赖硬规则

## 七、未解决问题

- **GitHub push 认证失败**（错误 056，全局档案）：本地已领先 origin/main 3 个 commit 待推。用户需更新 GitHub 凭据（`git credential-manager configure` 或重新登录）后再 `git push origin main`
- github.com 历史不可达问题可能复发，push 失败时记录到全局错误档案
- .exe stdout 在外部进程调用时不输出（PyInstaller --onefile + --windowed 模式 GUI 应用 console handle 继承问题）——不影响用户体验
- 软件功能仍仅 echo 一条 + UI 空壳可跳转 + 模型下拉框检测 + 分组 + AI 气泡模型名，但选了模型后无真实对话（chat() 未实现）
- 设计规范 spacing/shadow 表是推测值（未调 skill --persist 补全），后续可升级
- 仅支持 Ollama 本地 + 3 家云端预置占位，未接任何云端 API（留待 ADR-020+ 扩展）
- 模型选择未持久化（QSettings 落盘是下一候选需求）
