# Q-agent 临时记忆

更新时间：2026-06-20 12:30

## 一、项目详情

- 项目名：Q-agent
- 路径：G:\agent
- 仓库：https://github.com/zhihuzheye4/Q-agent
- 语言：Python 3.10+
- 定位：类似 Claude Code 的桌面端 AI 工具，纯软件形式对接本地 LLM
- 五层结构：对话/交互 → 编排/规划 → 记忆 → 技能/工具 → 工具调用层
- 关键约束：永不沙箱（仅基本输入校验）、本地 LLM 优先、零第三方依赖起步、每里程碑打 .exe、分发零外部依赖硬规则、安装包只保留最新一代

## 二、上次讨论

用户准确指出 v0.0.6 之前的分类 bug：
- minimax-m3 / deepseek-v4-pro 等通过 Ollama Cloud 转发的模型并非真正本地装权重

- 同时澄清：CLOUD_PRESET 是硬编码占位（ADR-019），但本地组是动态调 API（删除模型下次启动能检测出来）

## 七、未解决问题

- github.com 历史不可达问题可能复发，push 失败时记录到全局错误档案
- .exe stdout 在外部进程调用时不输出（PyInstaller --onefile + --windowed GUI 应用 console handle 继承）——不影响用户体验
- 软件功能仍仅 echo + UI 跳转 + 模型下拉框三组分类检测 + AI 气泡模型名 + llm 对称骨架（chat/complete 仍 NotImplementedError）
- 选了模型后无真实对话（chat() 未实现）——下一候选需求
- 设计规范 spacing/shadow 表是推测值
- CLOUD_PRESET 仍是硬编码占位，未接 OpenAI/Anthropic/Google API（cloud stubs 仅骨架）
- 模型选择未持久化（QSettings 落盘是下一候选需求）
- Ollama Cloud 判定依赖 remote_model/remote_host 字段，老版 Ollama 可能不返回，退化为 v0.0.6 行为（graceful，非问题）
