# Q-agent 临时记忆

更新时间：2026-06-20 13:00

## 一、项目详情

- 项目名：Q-agent
- 路径：G:\agent
- 仓库：https://github.com/zhihuzheye4/Q-agent
- 语言：Python 3.10+
- 定位：类似 Claude Code 的桌面端 AI 工具，纯软件形式对接本地 LLM
- 五层结构：对话/交互 → 编排/规划 → 记忆 → 技能/工具 → 工具调用层
- 关键约束：永不沙箱（仅基本输入校验）、本地 LLM 优先、零第三方依赖起步、每里程碑打 .exe、分发零外部依赖硬规则、安装包只保留最新一代
- 当前阶段：v0.0.9 完成（内存释放 + 切换模型清空上下文 + 模型名 hash 颜色），等待用户提下一个需求

## 二、上次讨论

用户提三个新功能需求：
1. 内存释放：不想和某模型对话时它仍在 RAM 待机占内存，需主动释放按钮（"释放当前模型内存"），下次需要时在下拉框重新选择即可加载
2. 切换模型后清空当前对话窗口上下文 + 提示"模型已切换为当前***"
3. 模型名字用不同颜色展示（现在统一灰色无法区分）

用户对关键 UX 决策的回答（AskUserQuestion 答复）：
- 释放按钮位置：工具栏独立按钮（与刷新并排，推荐）
- 颜色规则：按模型名 hash 取调色板（8 色，同一模型每次同色，推荐）
- 切换提示样式：居中灰色斜体系统提示（推荐）

## 七、未解决问题

- github.com 历史不可达问题可能复发，push 失败时记录到全局错误档案
- .exe stdout 在外部进程调用时不输出（PyInstaller --onefile + --windowed GUI 应用 console handle 继承）——不影响用户体验
- 取消按钮未接 ChatWorker.stop()（接口已留，v0.0.9+ 接）
- 历史消息首版全传（token 消耗大但简单），上下文裁剪留 v0.0.10+
- host 仍 hardcoded "http://localhost:11434"，QSettings 持久化留 v0.0.10+
- 设计规范 spacing/shadow 表是推测值
- CLOUD_PRESET 仍是硬编码占位，未接 OpenAI/Anthropic/Google API（cloud stubs 仅骨架）
- 模型选择未持久化（QSettings 落盘是下一候选需求）
- Ollama Cloud 判定依赖 remote_model/remote_host 字段，老版 Ollama 可能不返回，退化为 v0.0.6 行为（graceful，非问题）
- release_model 调用时如果模型正在 chat 生成中，Ollama 在当前生成完成后立即卸载（keep_alive=0 不强制中断当前生成）
- 切换模型清空对话是硬规则，不保留历史；用户切回原模型需重新输入
- 模型名 hash 颜色可能碰撞（crc32 mod 8 同值），8 色调色板日常足够，碰撞概率 1/8 可接受
- 新建对话/清空按钮仍是活 UI 空壳，未接 chat_page._clear_messages
