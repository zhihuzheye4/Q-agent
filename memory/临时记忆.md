## 一、项目详情

- 项目名：Q-agent
- 路径：G:\agent

## 二、上次讨论

用户要求："增加下一功能，在左侧工具栏，增加硬件占用曲线，CPU和GPU，实时显示"

通过 AskUserQuestion 确认 4 个维度：

## 三、上次执行

### 代码改动

- 新建 `q_agent/ui/hardware_monitor.py`：
  - `HardwareMonitor(QWidget)` 自定义 `paintEvent`：顶部 20px 图例（色块 + 名称 + 当前数值）+ 下方 140px 折线图（4 条叠加，0-100% y 轴，25%/50%/75% 网格线）
  - `HardwareMonitorWorker(QThread)` 1s 采集一次样本，`sample_collected(dict)` 信号回主线程
  - 4 指标 4 色：CPU 蓝 #3B82F6 / GPU 绿 #22C55E / VRAM 紫 #A855F7 / RAM 橙 #F97316
  - 容错：pynvml 初始化失败 → `_nvml_ok=False`，GPU/VRAM 永远 None，折线画灰色 #475569 N/A 占位横线
  - `None` 段断开不连线（避免错误跨断点连线）
  - `_on_sample` 追加历史 + 截断到 60s + `self.update()` 触发重绘
  - `collect_sample_sync()` 测试辅助函数
  - `contextlib.suppress(Exception)` 替代 `try-except-pass`（ruff 规则）

- 改造 `q_agent/ui/sidebar.py`：
  - 从 `QListWidget` 直接子类化改为 `QFrame` 容器（含 `QListWidget` tab 列表 + `HardwareMonitor` 底部硬件监控）
  - 保留 `tab_changed(int)` 信号、`SIDEBAR_WIDTH=200`、4 tab 元数据
  - `hardware_monitor` 作为公开属性供 MainWindow 调用 `start()` / `stop()`

- 改造 `q_agent/ui/main_window.py`：
  - `QTimer.singleShot(200, self.sidebar.hardware_monitor.start)` 启动 worker（延迟 200ms 让 UI 先绘制）
  - `closeEvent(event: QCloseEvent)` 调 `worker.stop()` + `wait(2000)` 优雅退出，避免线程悬挂
  - import `QCloseEvent` 替代 `# noqa: ANN001`（mypy strict 要求类型注解）

## 七、未解决问题

- github.com push 网络问题（v0.0.11 push 成功了 408787a..6bfc4e2，错误 057 自愈；但 v0.0.12 待 push，可能再触发）
- .exe stdout 在外部进程调用时不输出（PyInstaller --onefile + --windowed GUI 应用 console handle 继承）——不影响用户体验
- 取消按钮未接 ChatWorker.stop()（接口已留，v0.0.13+ 接）
- 历史消息首版全传（token 消耗大但简单），上下文裁剪留 v0.0.13+
- host 仍 hardcoded "http://localhost:11434"，QSettings 持久化留 v0.0.13+
- 设计规范 spacing/shadow 表是推测值
- CLOUD_PRESET 仍是硬编码占位，未接 OpenAI/Anthropic/Google API（cloud stubs 仅骨架）
- 模型选择未持久化（QSettings 落盘是下一候选需求）
- Ollama Cloud 判定依赖 remote_model/remote_host 字段，老版 Ollama 可能不返回，退化为 v0.0.6 行为（graceful，非问题）
- release_model 调用时如果模型正在 chat 生成中，Ollama 在当前生成完成后立即卸载（keep_alive=0 不强制中断当前生成）；v0.0.11 验证步骤会检测到模型仍在 → 抛错提示"可能正在生成中，请稍后重试"
- 切换模型清空对话是硬规则，不保留历史；用户切回原模型需重新输入
- 模型名 hash 颜色可能碰撞（crc32 mod 8 同值），8 色调色板日常足够，碰撞概率 1/8 可接受
- 新建对话/清空按钮仍是活 UI 空壳，未接 chat_page._clear_messages
- LoadingDots 固定 dot_size=8 不随 DPI 缩放（高 DPI 屏可能偏小）
- LoadingDots 流动彩虹色相独立于模型色，与"模型名 hash 色"色彩系统并行（设计取舍，非冲突）
- **Ollama 进程级 CUDA context 不立即归还 OS**：release_model /api/ps 验证通过 + nvidia-smi VRAM 已归还，但任务管理器看 ollama.exe 进程的"专用 GPU 内存"列可能仍显示几 GB 占用，这是 Ollama 已知行为非项目 bug，状态栏文案已明确告知用户
- **硬件监控无 NVIDIA 显卡时 GPU/VRAM 永远 N/A**：pynvml 容错降级，AMD/Intel GPU 不支持（需引入额外依赖，等用户提需求）
- **硬件监控采样间隔固定 1s 不可配置**：未来可在设置中调 1s/2s/5s
- **硬件监控历史窗口固定 60s 不可配置**：未来可调 60s/120s/300s
- **硬件监控不写盘持久化**：关闭重启后历史清空
