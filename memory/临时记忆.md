# Q-agent 临时记忆

更新时间：2026-06-20 12:30

## 一、项目详情

- 项目名：Q-agent
- 路径：G:\agent
- 仓库：https://github.com/zhihuzheye4/Q-agent
- 语言：Python 3.10+
- 定位：类似 Claude Code 的桌面端 AI 工具，纯软件形式对接本地 LLM
- 五层结构：对话/交互 → 编排/规划 → 记忆 → 技能/工具 → 工具调用层
- 关键约束：永不沙箱（仅基本输入校验）、本地 LLM 优先、零第三方依赖起步、每里程碑打 .exe、分发零外部依赖硬规则、安装包只保留最新一代
- 当前阶段：v0.0.8 完成（通过 Ollama 真实唤醒模型 + 流式批量刷新 500字/500ms 阈值），等待用户提下一个需求

## 二、上次讨论

用户提"既然模型已经检索并展现，那么下一步就是通过 Ollama 唤醒"。
明确 UX 决策："攒一段信息，比如500字，然后流失返回，这样可以提升出现速度，如果真就http实时监视的话，有些计算比较慢的，那真就是一秒钟一个字，用户会崩溃的"——拒绝纯 token 流式（一秒一字让人崩溃），要批量刷新。
云端预置（gpt-4o/claude/gemini，未接 API）选中时禁用发送按钮 + tooltip 提示。
