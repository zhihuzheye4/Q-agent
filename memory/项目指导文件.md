# Q-agent 项目指导文件

## 项目最终形态

类似 Claude Code 的桌面端 AI 工具：
- 纯软件形式（不是 Agent 框架 SDK）
- 对接本地 LLM 大模型
- 让本地 LLM 发出的指令被软件理解并执行
- 实现各大 AI 工具可实现的功能（对话、工具调用、技能学习、记忆等）

## 关键约束

- **永不沙箱**：本项目不引入沙箱模块。LLM 指令通过工具调用层直接执行，仅保留基本输入校验（危险命令黑名单 + 项目根目录保护）
- **本地 LLM 优先**：默认走本地后端，云端按需启用
- **零第三方依赖起步**：CLI 用 argparse，不引入 Typer/Click
- **分发零外部依赖硬规则**：最终用户拿 `.exe` 双击即跑，禁止让用户装任何外部依赖。所有运行时依赖由 PyInstaller `--onefile` 打包进单个 `.exe`（详见 CLAUDE.md 第二十节 + ADR-015）

## 技术栈

- 语言：Python 3.10+
- 包管理：pip + pyproject.toml（不引入 poetry/uv）
- 工具链：ruff（lint+format）+ mypy strict + pytest + pytest-cov
- 测试覆盖率：起步 30%，渐进到 70%

## 开发流程（增量盖房）

1. 用户提出一个具体需求
2. 讨论（可能多轮）→ 达成共识
3. AI 生成具体可执行方案 → 写入 `memory/方案/方案_YYYY-MM-DD_HHmm_主题.md`（L1 不可读）
4. 用户审阅方案
5. 用户确认后 → AI 执行（基于当前会话上下文，不回读方案文件）
6. 执行完毕 → 方案归档到 `memory/方案/已执行/`
7. 更新本文件（当前节点 + 已完成节点）
