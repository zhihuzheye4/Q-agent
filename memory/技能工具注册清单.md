## 工具层（Tools）— LLM 调用的底层操作

### list_models（Ollama）

- **名称**：list_models
- **描述**：查询本地 Ollama 服务可用模型列表（含本地/云端转发判定）
- **签名**：`list_models(host: str = "http://localhost:11434", timeout: float = 2.0) -> list[ModelEntry]`
- **模块**：`q_agent.llm.ollama`
- **返回**：`ModelEntry(name, is_remote, remote_host)` 列表。is_remote=True 表示该模型是 Ollama Cloud 转发的（非本地权重）
- **判定逻辑**：`is_remote = bool(remote_model) or bool(remote_host)`——Ollama /api/tags 的 remote_model / remote_host 字段任一非空即 cloud 转发
- **异常**：`OllamaError`（含面向用户的友好消息，覆盖连接拒绝/超时/HTTP 错误/JSON 解析失败）
- **依赖**：标准库 `urllib.request`，零新运行时依赖
- **状态**：已实现（v0.0.7 破坏性改签名 list[str] → list[ModelEntry]），UI 工具栏模型下拉框启动时调用
- **调用统计**：暂无

### release_model（v0.0.9 新增）

- **名称**：release_model
- **描述**：释放指定模型的内存（卸载出 Ollama 进程 RAM），不删除模型文件
- **签名**：`release_model(model: str, host: str = "http://localhost:11434", timeout: float = 30.0) -> None`
- **模块**：`q_agent.llm.ollama`
- **实现**：POST /api/generate body={"model":..., "keep_alive": 0} 触发 Ollama 在当前生成完成后立即卸载模型权重；模型不在内存时是 no-op
- **异常**：`OllamaError`（HTTPError / URLError / TimeoutError 覆盖连接拒绝/超时/HTTP 错误，复用 list_models 的 urllib 错误处理模式）
- **依赖**：标准库 `urllib.request` + `json`，零新运行时依赖
- **状态**：已实现，UI 工具栏 release_btn 触发 ModelReleaseWorker 后台调用
