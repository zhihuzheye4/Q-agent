## 工具层（Tools）— LLM 调用的底层操作

### list_models（Ollama）

- **名称**：list_models
- **描述**：查询本地 Ollama 服务可用模型列表（含本地/云端转发判定）
- **签名**：`list_models(host: str = "http://localhost:11434", timeout: float = 2.0) -> list[ModelEntry]`
- **模块**：`q_agent.llm.ollama`
- **返回**：`ModelEntry(name, is_remote, remote_host)` 列表。is_remote=True 表示该模型是 Ollama Cloud 转发的（非本地权重）
- **判定逻辑**：`is_remote = bool(remote_model) or bool(remote_host)`——Ollama /api/tags 的 remote_model / remote_host 字段任一非空即 cloud 转发
- **异常**：`OllamaError`（含面向用户的友好消息，覆盖连接拒绝/超时/HTTP 错误/JSON 解析失败）
- **依赖**：标准库 `urllib.request`，零新运行时依赖
- **状态**：已实现（v0.0.7 破坏性改签名 list[str] → list[ModelEntry]），UI 工具栏模型下拉框启动时调用
- **调用统计**：暂无

### OllamaClient（v0.0.6 新增，llm 层对称骨架）

- **类**：`q_agent.llm.ollama.OllamaClient(LLMClient)`
- **构造**：`OllamaClient(model: str, host: str = "http://localhost:11434")`

## UI 组件（不属于技能/工具，供编排层与 UI 协作参考）

### Toolbar 模型下拉框（v0.0.7 起本地/Ollama Cloud/云端预置三组）

- **类**：`q_agent.ui.toolbar.Toolbar`
- **信号**：`model_selected(str)` — 用户切换模型时 emit 模型名
- **方法**：`refresh_models()` 触发异步检测；`current_model() -> str | None` 取当前选中
- **下拉结构（三组）**：
  - `本地（Ollama）` header + is_remote=False 的真本地模型项（无则占位"未发现本地模型"）
  - `Ollama Cloud（转发）` header + is_remote=True 的转发模型项（仅存在转发模型时才显示该组）
  - `云端（占位，未接 API）` header + CLOUD_PRESET 3 家预置（gpt-4o/claude-opus-4-7/gemini-2.5-pro，未接 API）
- **协作**：`ModelRefreshWorker(QThread)` 后台跑 list_models，信号 `models_found(list)` / `refresh_failed(str)` 回主线程

### ChatPage 对话页（v0.0.5 起 AI 气泡带模型名标签）

- **类**：`q_agent.ui.pages.chat_page.ChatPage`

- **AI 气泡布局**：`[垂直列：模型名 QLabel#ModelLabel（11px 灰） + 气泡 QLabel#MessageAI][stretch]`

## 开发辅助脚本（dev-only，不入分发包）

> 这些脚本在开发期辅助生成代码/资源，本身不在 `q_agent` 包内，不会进 `.exe`。

### generate_icons

- **路径**：`scripts/generate_icons.py`
- **用途**：UI 矢量图生成（SVG sprite + 可选 PNG + manifest.json）
- **调用方式**：
  - `python scripts/generate_icons.py`（默认生成 SVG）
  - `python scripts/generate_icons.py --png`（按需，转 PNG 用 PySide6.QtSvg）
  - `python scripts/generate_icons.py --clean`（清理测试产物）
- **技术栈**：PySide6 + QtSvg（延迟 import，仅 `--png` 时需）
- **设计哲学**：预制调用而非实时渲染；PNG 转换按需使用；测试产物用完即删
- **状态**：已实现，5 级递进复杂度测试通过 + 6 个 UI 用图标（chat/skills/memory/new-chat/clear/about）已生成
- **当前产物**：11 个 SVG + manifest.json（含 5 个测试样本 + 6 个 UI 实用图标）

### ChatWorker（v0.0.8 新增）

- **路径**：`q_agent/ui/chat_worker.py`
- **用途**：后台跑 OllamaClient.chat_stream，批量 flush chunk 到主线程（避免一秒一字崩溃）
- **状态**：已实现
- **关键设计**：
  - QThread + 3 信号：`chunk_received(str)` / `chat_failed(str)` / `chat_done()`
  - CHUNK_SIZE=500 / FLUSH_MS=500.0 混合刷新阈值（buffer 攒满 500字 OR 距上次 flush 满 500ms 任一触发）
  - stop() 接口留作未来取消按钮（v0.0.8 未接 UI）
  - 异常捕获 OllamaError + Exception → emit chat_failed

### ChatPage 对话流（v0.0.8 接真实调用）

- **路径**：`q_agent/ui/pages/chat_page.py`
- **用途**：发送对话实际调 Ollama /api/chat 流式拿回复显示在 AI 气泡
- **状态**：已实现（取代 v0.0.1~v0.0.7 的 echo 占位）
- **关键设计**：
  - `SENDABLE_GROUPS = ("local", "ollama-cloud")`：仅本地 + Ollama Cloud 转发可发送，云端预置（cloud，未接 API）禁用 + tooltip
  - `_add_message` 加 `append_to_history: bool` 参数，pending AI 气泡用 False 不污染历史
  - `_on_send_clicked`：取 group 防御 → add user → 构造 messages_for_llm → pending AI 气泡 → loading 状态（禁用输入+按钮变"生成中"）→ 启动 ChatWorker
  - 3 slot：`_on_chunk`（追加到 pending 气泡）/ `_on_chat_failed`（pending 气泡 objectName 改 MessageError 红色）/ `_on_chat_done`（完整回复入 _messages + 清 pending）
  - `update_send_enabled(group)`：cloud→禁用 + tooltip / SENDABLE_GROUPS→has_text 判断 / None→禁用
  - `_scroll_to_bottom` 加 shiboken6.isValid 防御 + contextlib.suppress(RuntimeError)（测试跨用例残留 deferred 调用崩）

### OllamaClient.chat_stream（v0.0.8 新增）

- **路径**：`q_agent/llm/ollama.py`
- **用途**：流式调 POST /api/chat 拿 chunk 文本（取代 v0.0.6 的 NotImplementedError）
- **状态**：已实现
- **关键设计**：
  - NDJSON 响应逐行 json.loads，yield `chunk.message.content`
  - 遇 `chunk.done=true` 返回（先 yield 剩余 content 再 return）
  - 错误处理复用 list_models urllib 模式（HTTPError 在 URLError 之前捕获，socket.timeout 单独处理）
  - chat() = "".join(chat_stream())，同步便捷方法供编排层非流式调用
