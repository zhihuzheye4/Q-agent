### [AI 答 1：PySide6 和 PyQt5 关系]
两者都是 Python 绑定 Qt 框架的库，同源不同主。PyQt5 = 英国 Riverbank 第三方做的 Qt 5 绑定，GPL 协议商用需付费；PySide6 = Qt Company 官方做的 Qt 6 绑定，LGPL 协议商用免费。API 约 99% 一致，仅 import 名不同（PyQt5.QtWidgets vs PySide6.QtWidgets）。会一个就会另一个，迁移基本就是改 import。商用选 PySide6（LGPL），学习资源 PyQt5 多（历史久）。

### [AI 答 2：token 消耗对比]
调 /ui-ux-pro-max skill 设计 UI 的 token 远多于调脚本，差一到两个数量级。skill 是"AI 即时服务"——skill 内容加载到上下文 + AI 阅读 + AI 生成设计系统 + AI 多轮搜索补充 + AI 据此写代码，全程烧 token，一次 --design-system 输出可能 5K-20K token，加上 --domain 多次搜索和后续讨论轻松破十万。脚本是"AI 写一次，机器跑 N 次"——AI 只在写脚本时烧 token（几千），之后 python scripts/generate_icons.py 是本地 Python 执行，AI 只看几十行输出几乎不烧 token。这跟"预制矢量图调用 vs 实时渲染"是同一原理：预制片省运行时成本，脚本预制省 AI token。

### [AI 答 3：skill 角色定位]
主要作用：教如何设计 UI。次要作用：能生成部分图标设计稿，但不是批量生成项目调用的图标文件。skill 输出形态是 markdown 设计规范文档 / 设计稿 / 代码片段。子 skill 含 design-system（设计系统）、--domain（领域最佳实践）、--stack（技术栈最佳实践）、banner-design（横幅）、design（综合设计含图标）、ui-styling（shadcn+Tailwind）。正确分工：skill 给"规范"（一次性、指导性），脚本按"规范"生成"产物"（可重复跑、执行性）。两者互补不替代。

### [AI 答 1：Qt5 vs Qt6 缺失对比]
没有核心功能是 Qt5 能做而 Qt6 做不到的。Qt6 是 Qt5 升级版，绝大部分 API 兼容且多为超集。Qt6 相对 Qt5 优势：默认高 DPI、Qt Quick 3D 完整支持、C++17/20 现代基线、图形渲染性能优化（Vulkan/Metal/D3D 后端）、新模块（Coap/Quick3DPhysical 等）。Qt6 移除的极少数老旧模块（Qt Script、Qt XmlPatterns）都是早已弃用的边缘模块，不影响正常 GUI 开发。哪个更强大：Qt6 更强大。Qt5 更成熟（生态/第三方兼容/文档资源）。长期看 Qt6 是未来——Qt 5.15 是 Qt5 最后 LTS，主线已转 Qt6。Python 绑定含义：PyQt5=绑定 Qt5（过去式），PySide6=绑定 Qt6（官方主推未来）。选 PySide6=选未来。

### [AI 答 2：工作流确认]
全部可以脚本全权完成。完整工作流：1) 调 /ui-ux-pro-max --design-system 产设计规范（markdown）；2) scripts/generate_icons.py 读规范生成 SVG（Python 直接写 SVG XML 字符串，SVG 是文本格式不需渲染引擎）；3) 同脚本或 scripts/svg_to_png.py 转 SVG→PNG（多技术选项）；4) UI 加载时调用预制图标不实时渲染。SVG→PNG 转换选项：PySide6.QtSvg（QSvgRenderer→QImage→PNG，零额外依赖，技术栈选 PySide6 的隐形收益）、cairosvg（轻量纯 Python 调 cairo）、Pillow+svglib（svglib 对复杂 SVG 支持有限）、inkscape 命令行（高保真但依赖外部软件）、wand/ImageMagick（同上）。关键结论：选 PySide6 可复用 QtSvg 模块做转换零额外依赖；选 Tkinter 需引入 cairosvg 或 Pillow 作为 dev 依赖。
