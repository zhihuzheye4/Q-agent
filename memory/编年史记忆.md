### 修复内容

- **问题**：用户双击 安装包/v0.0.1/Q-agent.exe 报错：
  `Failed to execute script 'cli' due to unhandled exception:'NoneType' object has no attribute 'write'`
  完整 traceback 指向 argparse 在无参数时尝试写 usage 到 stderr
- **根因**：`--windowed` 模式下 sys.stderr 是 None（无 console），argparse.required=True 子命令无参数时调用 error→print_usage→_print_message→sys.stderr.write，而 sys.stderr=None 导致 AttributeError
- **修复**：cli.py 的 main() 检测无参数时默认设 argv=["ui"]，自动启动 UI（GUI 应用标准行为——双击 .exe 直接进 UI）
- **验证**：重打 .exe 后双击（无参数）UI 正常启动 PID 12360 持续运行
- **教训**：PyInstaller --windowed GUI 应用 + argparse required 子命令 = 致命组合，必须默认有 fallback 行为；双击场景应自动进 GUI，不是报错
