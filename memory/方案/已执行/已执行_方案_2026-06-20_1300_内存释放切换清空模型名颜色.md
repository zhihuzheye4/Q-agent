# 已执行方案：模型内存释放 + 切换模型清空上下文 + 模型名 hash 颜色

执行时间：2026-06-20 13:00
版本：v0.0.9
对应 commit：38dc057

## 一、用户需求

> 给当前模型唤醒增加一个功能，即内存释放，也就是说我不想和某模型对话的时候，但是模型依然内存内加载着，待机状态，增加一个内存释放功能，当不需要使用模型的时候，让其处于为唤醒状态即可，当需要的时候在下拉框重新选择模型即可。
> 第二，切换模型以后清空当前对话窗口的上下文，并提示模型以切换当前***。
> 第三，模型名字用不同颜色展示。

## 二、UX 决策（AskUserQuestion 答复）

1. 释放按钮位置：**工具栏独立按钮**（与刷新并排，推荐）
2. 颜色规则：**按模型名 hash 取调色板**（8 色，同一模型每次同色，推荐）
3. 切换提示样式：**居中灰色斜体系统提示**（推荐）

## 七、设计取舍

- **释放按钮位置选工具栏而非下拉框旁边**：工具栏已有刷新按钮，释放与刷新属同层级管理操作，并排放视觉协调
- **hash 用 zlib.crc32 而非内置 hash()**：Python 内置 hash() 对字符串做了随机化（PYTHONHASHSEED），同一模型每次启动可能不同色；zlib.crc32 跨进程跨启动稳定
- **8 色调色板而非无限渐变**：8 个高对比色日常区分足够，碰撞概率 1/8 可接受；多于 16 色反而难分辨
- **切换模型清空是硬规则**：不保留历史。用户切回原模型需重新输入。原因：切换后历史对当前模型无意义（context 不同），保留易误导用户以为可继续
- **首次自动选择抑制 model_selected**：避免启动后自动选首个本地模型触发清空初始问候；仍 emit group_changed 同步发送按钮状态
- **release_model 调用时不强制中断当前生成**：keep_alive=0 让 Ollama 在当前生成完成后卸载，不强行打断；若需强中断接 ChatWorker.stop()（v0.0.10+ 候选）
