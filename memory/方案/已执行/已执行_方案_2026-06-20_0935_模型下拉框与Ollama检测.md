# 方案：UI 右上角模型下拉框 + 启动时检测本地 Ollama LLM

生成时间：2026-06-20 09:35
状态：待执行（讨论已与用户达成共识）

## 一、需求

用户：双击运行软件时，在软件右上方添加模型下拉框，软件启动后检查本地可用 LLM 大模型。这是第一个填充的实际功能。

## 二、决策（已与用户确认）

- 后端：只支持 Ollama（HTTP GET http://localhost:11434/api/tags，零新依赖）
- 检测失败 UX：下拉框显示"未发现本地 LLM" + 旁边刷新按钮，状态栏同步提示

## 三、技术设计

### 3.1 新增 q_agent/llm/ollama.py

- `list_models(host: str = "http://localhost:11434", timeout: float = 2.0) -> list[str]`
  - urllib.request.urlopen（标准库，PyInstaller 友好，零新运行时依赖）
  - GET `/api/tags`，解析 JSON `{"models": [{"name": "qwen2.5:7b"}, ...]}`
  - 失败抛 `OllamaError`（含友好消息：连接拒绝 / 超时 / HTTP 错误 / JSON 解析失败）
- `OllamaError(Exception)` 自定义异常
- 不实现 `chat()`（下一阶段才接对话能力）

### 3.2 新增 refresh 图标

- generate_icons.py 加 `icon_refresh_active`（Feather/Lucide refresh-cw 风格，双圆弧箭头）
- 注册到 ICONS 列表
- 跑 `python scripts/generate_icons.py` 生成 refresh-active.svg + 更新 manifest.json

### 3.3 改造 q_agent/ui/toolbar.py

当前：左对齐 3 个 QToolButton（新建/清空/关于）。
改造后：
- 左侧保留 3 个按钮
- 右侧 addWidget(new)：模型标签 "模型:" + QComboBox + 刷新 QToolButton
- 用 addSeparator() 分隔左右两组
- 右侧用 QWidget 包装（QHBoxLayout），靠右对齐

### 3.4 异步检测（QThread + Signal）

避免网络请求阻塞 UI 主线程：
- `ModelRefreshWorker(QRunnable)` 或 QThread 子类
- 信号 `models_found(list[str])` / `refresh_failed(str)`
- toolbar 提供方法 `refresh_models()`：启动 worker，下拉框临时显示"检测中..."
- MainWindow 启动后 `QTimer.singleShot(100, toolbar.refresh_models)` 自动触发一次

### 3.5 状态显示

- 检测中：下拉框只有一项 "检测中..."
- 成功有模型：下拉框填模型名列表
- 成功无模型：下拉框显示"未发现本地 LLM"（仅 1 项）
- 失败（连接拒绝/超时）：下拉框显示"未发现本地 LLM"，状态栏 "未发现本地 LLM，请启动 Ollama 服务后点击刷新"

### 3.6 测试

- tests/test_llm_ollama.py：mock urllib.request.urlopen
  - happy：返回 `{"models":[{"name":"qwen2.5:7b"},{"name":"llama3:8b"}]}` → `["qwen2.5:7b", "llama3:8b"]`
  - 连接拒绝：urlopen 抛 URLError → OllamaError
  - 超时：urlopen 抛 socket.timeout → OllamaError
  - HTTP 错误：HTTPError → OllamaError
  - JSON 解析失败：返回非 JSON → OllamaError
- tests/test_ui_toolbar.py：测 ModelRefreshWorker 信号触发（用 Qt test）

## 五、风险与权衡

- 异步检测用 QThread 而非 QNetworkAccessManager：QThread+urllib 更简单，QNetworkAccessManager 在 PyInstaller --onefile 下需额外 Qt Network 模块（已含但更复杂）
- 检测超时 2 秒：用户机器若 Ollama 启动慢可能误判未发现 → 提供手动刷新按钮兜底
- 仅 Ollama 不支持 LM Studio：用户确认第一版只支持一个，后续 ADR-019 再扩
- 不引入 requests 库：坚守分发零外部依赖硬规则（CLAUDE.md 第二十节）

## 六、不做什么

- 不实现 chat() 调用（下一阶段）
- 不持久化用户选择的模型（QSettings 落盘是下一个候选需求）
- 不做 Ollama 服务状态轮询（仅启动 + 手动刷新）
- 不做模型详情展示（参数 / 大小 / 上下文长度）——下拉框只列名字
