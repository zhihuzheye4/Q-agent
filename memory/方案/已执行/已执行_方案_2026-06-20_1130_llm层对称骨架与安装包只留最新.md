# 方案：llm 层对称骨架 + 安装包只保留最新

时间：2026-06-20 11:30
状态：已执行（commit 5cff4c3 + fb45de8）

## 背景

用户三问：
1. 软件包不需要保留每一代，只需要保留最新一代即可
2. 当前软件并没有区分云端和本地，或你编写的 py 文件并没有区分云端和本地的能力？
3. 是否严格遵循模块化？为什么不先完成最小闭环再添加枝叶？

核心问题：v0.0.4 时 ollama.py 仅作为检测函数（list_models + NotImplementedError chat/complete），未纳入 LLMClient 体系；v0.0.5 的 UI 分组只是壳，后端没有对称的 client 类层级。

## 决策

1. **llm 层对称骨架**：
   - `q_agent/llm/ollama.py` 升级：新增 `OllamaClient(LLMClient)` 子类，构造 (model, host) + chat/complete 抛 NotImplementedError
   - `q_agent/llm/cloud/` 新建子包：openai.py + anthropic.py + gemini.py 三个 stub，构造 (model, api_key, base_url) + chat/complete 抛 NotImplementedError
   - `q_agent/llm/__init__.py` 统一导出全部 client
   - 后端分类清晰：local（Ollama + llama.cpp 占位）+ cloud（OpenAI / Anthropic / Gemini）
2. **所有 client 构造成功，chat/complete 抛 NotImplementedError**：下一阶段接真 API 时只填方法体，不动构造与接口
3. **依赖管理坚守 ADR-015**：urllib 标准库零新运行时依赖
4. **安装包只保留最新**：每次打新版本 .exe 删掉上一版子目录，只留当前最新；v0.0.1~v0.0.4 已删
5. **CLAUDE.md §17 同步更新**
