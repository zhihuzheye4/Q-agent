# 方案：通过 Ollama 唤醒模型 + 流式批量刷新（v0.0.8）

- **日期**：2026-06-20
- **状态**：已执行
- **对应 commit**：10ae75e
- **对应 ADR**：ADR-022

## 背景

v0.0.7 完成模型下拉框三组分类后，下一步是把已选模型真正用起来——用户发送对话，实际调 Ollama /api/chat 拿回复显示在 AI 气泡。

当前 _on_send_clicked（q_agent/ui/pages/chat_page.py）是 echo 占位，不调 LLM。OllamaClient.chat() 抛 NotImplementedError。

用户明确 UX 决策：
- 响应方式：流式 + 批量刷新——攒够 500字 OR 满 500ms 任一触发就 flush 一段到气泡。避免纯 token 流式"一秒一字让人崩溃"
- 云端预置占位（gpt-4o/claude/gemini，未接 API）选中时禁用发送按钮 + tooltip 提示

## 实施内容

### 一、后端：填充 OllamaClient（q_agent/llm/ollama.py）

新增 `chat_stream(messages, timeout=120.0) -> Iterator[str]`：流式调 POST /api/chat，stream=true。响应是 NDJSON（每行一个 chunk），逐行 json.loads，yield chunk.message.content，遇 done=true 返回。错误处理复用 list_models 的 urllib 模式（HTTPError 在 URLError 之前捕获，socket.timeout 单独处理）。

chat(messages) -> str 填充为 "".join(self.chat_stream(messages))——同步便捷方法，供编排层非流式调用。

### 二、UI worker：新建 q_agent/ui/chat_worker.py

ChatWorker(QThread)：
- 信号：chunk_received(str) / chat_failed(str) / chat_done()
- 常量：CHUNK_SIZE = 500，FLUSH_MS = 500.0
- run()：创建 OllamaClient(model, host)，遍历 chat_stream()，累积 buffer + 记 last_flush = time.monotonic()；每 chunk 来时检查 len(buffer) >= 500 OR (now - last_flush) * 1000 >= 500 → emit chunk_received(buffer) + 清空 buffer + 重置 last_flush。循环结束 emit 剩余 buffer + chat_done()
- stop()：设 self._stop = True，run 内每次循环检查早退（首版未接入 UI 取消按钮，但留接口）
- 异常捕获 OllamaError / Exception，emit chat_failed(str(e))

### 三、toolbar 暴露 group（q_agent/ui/toolbar.py）

- 新增 model_group_changed = Signal(object) 信号（str | None）
- _on_combo_changed emit model_selected(text) 后追加 emit model_group_changed(item.data(ITEM_ROLE))
- 新增公开方法 current_model_group() -> str | None：取当前选中项的 ITEM_ROLE 数据

### 四、chat_page 接入真实调用（q_agent/ui/pages/chat_page.py）

新增常量：SENDABLE_GROUPS = ("local", "ollama-cloud")——只有本地和 Ollama Cloud 转发可发送。

_add_message 改造：加可选参数 append_to_history: bool = True，返回 bubble_label 引用供外部追加 text。pending AI 气泡用 append_to_history=False 不污染 _messages。

_on_send_clicked 重写：
1. strip 输入，空返回
2. 取 group = self._group_provider()（新增注入）；非 SENDABLE_GROUPS 防御性返回
3. model_name = self._current_model_name()
4. _add_message("user", text) → _messages 加 user
5. messages_for_llm = [{"role": r, "content": t} for r, t, _ in self._messages]
6. pending_bubble = self._add_message("ai", "", model_name=model_name, append_to_history=False)
7. loading 状态：send_btn.setEnabled(False) + input.setEnabled(False) + send_btn.setText("生成中")
8. self.input.clear() + _scroll_to_bottom()
9. 创建 ChatWorker(model_name, self._host, messages_for_llm, parent=self)
10. 连接 chunk_received → _on_chunk / chat_failed → _on_chat_failed / chat_done → _on_chat_done
11. self._worker.start()

新增 slot：
- _on_chunk(text)：self._pending_text += text + self._pending_bubble.setText(self._pending_text) + _scroll_to_bottom()
- _on_chat_failed(msg)：把 pending_bubble 文本设为 "❌ {msg}"，objectName 改 "MessageError"（theme 新增红色样式），_reset_send_state()
- _on_chat_done()：self._messages.append(("ai", self._pending_text, model_name))，清 pending 状态，_reset_send_state()

新增方法：
- set_group_provider(provider: Callable[[], str | None])
- set_host(host: str)
- update_send_enabled(group: str | None)：cloud→禁用 + tooltip / SENDABLE_GROUPS→has_text 判断 / None→禁用

_on_input_changed 改为调 update_send_enabled(group)，避免 group=cloud 时被绕过禁用。

_scroll_to_bottom 加 shiboken6.isValid 防御 + contextlib.suppress(RuntimeError)，修测试跨用例残留 deferred 调用崩。

### 五、main_window 连接（q_agent/ui/main_window.py）

- 注入 self.chat_page.set_group_provider(self.toolbar.current_model_group) + self.chat_page.set_host("http://localhost:11434")
- 连接 self.toolbar.model_group_changed.connect(self.chat_page.update_send_enabled)
- 启动时主动调一次：QTimer.singleShot(150, self._sync_send_enabled)

### 六、theme 错误气泡样式（q_agent/ui/theme.py）

新增 QLabel#MessageError 红字红底红边，与正常 AI 气泡视觉区分。
