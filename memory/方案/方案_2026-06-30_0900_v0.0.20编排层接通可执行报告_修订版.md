## 二、Claude Code 调研结论 + Q-agent 适配决策

### 2.1 Plan 模式

**Claude Code 行为**：LLM 自主调 EnterPlanMode → 委托独立只读 Plan 子 agent → ExitPlanMode 需用户授权 → plan 文本不强制落盘 → 用户否决后 LLM 留原会话调整重提。

**Q-agent 适配**：**不派发独立 Plan 子 agent**——本地小模型不存在 Haiku 级快速模型，派发独立子 agent 反而拖慢。改为**父 agent 内同会话切权限态**：
- `enter_plan_mode` 工具调用后，TurnState 加 `plan_mode: bool = False` 字段
- `plan_mode=True` 时，dispatcher 拒绝所有 write/destructive 工具调用（直接回喂 PermissionError "plan 模式禁写"）
- `exit_plan_mode` 工具调用携带 plan 文本 → **新建 PlanApprovalDialog**（不复用 ApprovalDialog，语义不同：plan 是"是否执行此计划"，destructive 是"是否允许此破坏性操作"）→ 用户确认后 plan 文本落盘到 `~/.q-agent/plans/{session_id}/{ts}.md` + 退出 plan 模式
- 用户否决 plan → 留 plan 模式 + 把否决理由回喂 LLM 继续研究

### 2.2 子 Agent 派发（v0.0.20 不实施）

**Claude Code 行为**：LLM 显式调 Agent 工具 → 子 agent 独立干净上下文 → 父只收子最终文本 → 嵌套 5 层 / fork / worktree 隔离。

**Q-agent 适配方向（仅留档，v0.0.20 不实施）**：2026-06-30 用户明确指示"子 agent 暂时不考虑，后面会有单独开发环节"。v0.0.20 范围不含 Agent 子 agent 派发。适配思路：**砍掉嵌套 5 层/fork/worktree**——保留单层、独立上下文、最终文本回喂即可（详见后续单独里程碑方案）。

### 2.3 TodoWrite 任务管理

**Claude Code 行为**：v2.1.142 起 TodoWrite 默认禁用，改 TaskCreate/TaskGet/TaskList/TaskUpdate 四件套 → JSONL 落盘 → Ctrl+T 独立浮层 UI。

**Q-agent 适配**：**单一 TodoWrite 工具 + 会话内 JSON 持久化 + TypedTask 类型**——小模型调四件套过复杂：
- `todo_write(tasks: list[TodoTask], session_id: str) -> str`，TodoTask 为 TypedDict：`{subject: str, description: str, status: Literal["pending","in_progress","completed","deleted"], activeForm: str}`
- 持久化：会话内 JSON 落盘到 `~/.q-agent/todos/{session_id}.json`，切换会话保留（同会话内跨压缩保留，跨会话不共享）
- **JSON 而非 sqlite 的理由**：todo 列表是会话内独立工件，JSON 文件便于人工审计 + git diff 跟踪任务演进 + 跨会话独立文件不混淆；sessions 走 sqlite 是因为 sessions 间有外键关联（messages/tool_calls/compactions），todos 无此需求
- UI 展示：技能 tab 下方新增"任务"区域（QTableWidget 4 列），从 `~/.q-agent/todos/{session_id}.json` 加载
- LLM 调用时机：规划阶段一次性写入 + 执行中动态更新

### 2.4 Sessions_* 会话管理

**Claude Code 行为**：sessions 是用户命令（/resume、/clear、/compact、--resume）→ JSONL 落盘 → 切换是"保留+恢复"模型 → 子 agent 即新会话单元。

**Q-agent 适配**：**作为 LLM 可调工具 + sqlite 持久化**——Q-agent 哲学是"让 LLM 指令被软件理解执行"，sessions 锁死在用户命令层违背此哲学：
- 3 个工具：`sessions_list() / sessions_switch(session_id) / sessions_new()`
- 持久化：复用 v0.0.18 persistence.py 的 sessions 表（已建）
- 切换上下文：清空当前内存消息历史 + `load_messages(limit=N)` 加载新会话历史（与 Claude Code "保留+恢复"模型对齐——不删旧会话，可后续 switch 回来）
- UI：会话切换由 LLM 工具调用触发，UI 自动刷新消息流；未来 v0.0.21+ 可加用户命令 `/sessions` 列表选择

## 三、主体接通设计（M1-M3）

### 3.1 M1 主体接通（修订：核心改造）

#### 3.1.1 run_turn 改造：流式 chunk 回调 + cooperative cancellation

**问题**：v0.0.18 `loop.py:264` 的 `_call_llm` 用 `list(chat_stream(clean))` 一次性拼接完整文本才返回，**不流式 emit**；`run_turn` 的 while-true **无中断机制**。直接接通会丢 UI 流式体验 + v0.0.17 stop 按钮失效。

**改造方案**：

```python
# q_agent/orchestrator/loop.py（扩展 Orchestrator）
class Orchestrator:
    def __init__(
        self,
        llm_client: Any,
        context_manager: ContextManager,
        session_store: SessionStore,
        executor: Any,
        summary_worker: SummaryWorker | None = None,
        max_steps: int = 30,
        consecutive_error_threshold: int = 3,
        doom_loop_threshold: int = 3,
        on_step: Callable[[Any], None] | None = None,
        on_compaction: Callable[[int, int, int], None] | None = None,
        # v0.0.20 新增 2 参数
        on_chunk: Callable[[str], None] | None = None,  # 流式 chunk 回调
        on_tool_call: Callable[[str, str], None] | None = None,  # (tool_name, event) event="started"/"finished"
    ) -> None:
        ...
        self._cancel_event = threading.Event()  # cooperative cancellation

    def cancel(self) -> None:
        """主线程请求取消（ChatWorker.stop 调用）。"""
        self._cancel_event.set()

    def _is_cancelled(self) -> bool:
        return self._cancel_event.is_set()

    def _call_llm_streaming(self, messages: list[dict[str, object]]) -> str:
        """流式调 LLM，每个 chunk 通过 on_chunk 回调 emit 给 UI，同时拼接完整文本返回。"""
        chat_stream = getattr(self.llm_client, "chat_stream", None)
        if chat_stream is None:
            # 退化到非流式 chat
            chat = getattr(self.llm_client, "chat", None)
            if chat is None:
                raise RuntimeError("LLM 客户端未实现 chat 或 chat_stream 方法")
            result: str = chat(messages)
            if self.on_chunk is not None:
                self.on_chunk(result)  # 一次性 emit
            return result
        clean = [
            {"role": str(m["role"]), "content": str(m.get("content", ""))} for m in messages
        ]
        parts: list[str] = []
        for text in chat_stream(clean):
            if self._is_cancelled():
                # 取消：抛 LLMCancelled 让上层捕获，已收到的 chunk 已 emit
                raise _LLMCancelled()
            parts.append(text)
            if self.on_chunk is not None:
                self.on_chunk(text)
        return "".join(parts)
```

`run_turn` 主循环改动：
- 第 3 步 `_call_llm` → `_call_llm_streaming`
- while-true 每步开头检查 `if self._is_cancelled(): turn.mark_terminated(TerminationReason.CANCELLED); break`
- 第 6 步工具执行前 emit `on_tool_call(tool_name, "started")`，执行后 emit `on_tool_call(tool_name, "finished")`
- 捕获 `_LLMCancelled` → flush 已收到文本入历史 + break

```python
# q_agent/ui/chat_orchestrator_bridge.py（新建桥接层）
"""ChatPage 与 Orchestrator 的桥接层。

隔离编排层构造细节，chat_page 仅换一行 import + 一行构造。
避免 §21 模块化原则下侵入 chat_page 既有结构。
"""
from pathlib import Path
from typing import Any

from q_agent.orchestrator.context import ContextManager
from q_agent.orchestrator.loop import Orchestrator
from q_agent.orchestrator.persistence import SessionStore
from q_agent.orchestrator.summary_worker import SummaryWorker


def build_orchestrator(
    llm_client: Any,
    db_path: Path,
    session_id: str,
    summary_model: str = "qwen2.5:3b",
    on_chunk: Any = None,
    on_tool_call: Any = None,
) -> Orchestrator:
    """构造完整接通的 Orchestrator 实例。"""
    store = SessionStore(db_path)
    # 确保 session 存在
    if not store.session_exists(session_id):
        store.create_session(session_id)
    context_mgr = ContextManager(store, session_id)
    # 加载历史消息
    context_mgr.load_from_store(limit=200)
    summary_worker = SummaryWorker(model=summary_model, host="http://localhost:11434")
    return Orchestrator(
        llm_client=llm_client,
        context_manager=context_mgr,
        session_store=store,
        executor=_get_executor(),
        summary_worker=summary_worker,
        on_chunk=on_chunk,
        on_tool_call=on_tool_call,
    )


def _get_executor() -> Any:
    """获取工具执行器（q_agent.tools.executor 模块）。"""
    from q_agent.tools import executor
    return executor
```

#### 3.1.3 ChatWorker 适配

```python
# q_agent/ui/chat_worker.py（扩展）
class ChatWorker(QThread):
    chunk_received = Signal(str)
    chat_failed = Signal(str)
    chat_done = Signal()
    chat_aborted = Signal()
    # v0.0.20 新增 2 信号
    tool_call_started = Signal(str)
    tool_call_finished = Signal(str, str, str)  # (tool_name, status, output_path)

    def __init__(
        self,
        orchestrator: Orchestrator,  # 改为接收 Orchestrator 而非 model/messages
        user_text: str,
        parent: QWidget | None = None,
    ) -> None:
        super().__init__(parent)
        self._orchestrator = orchestrator
        self._user_text = user_text
        self._stop = False

    def run(self) -> None:
        try:
            # 注入 chunk 回调（emit 到主线程）
            self._orchestrator.on_chunk = self._emit_chunk
            self._orchestrator.on_tool_call = self._emit_tool_call
            result = self._orchestrator.run_turn(self._user_text)
            if result.termination_reason == TerminationReason.CANCELLED:
                self.chat_aborted.emit()
            else:
                self.chat_done.emit()
        except Exception as e:
            self.chat_failed.emit(f"编排层错误：{e}")

    def _emit_chunk(self, text: str) -> None:
        self.chunk_received.emit(text)

    def _emit_tool_call(self, tool_name: str, event: str) -> None:
        if event == "started":
            self.tool_call_started.emit(tool_name)
        else:
            self.tool_call_finished.emit(tool_name, event, "")

    def stop(self) -> None:
        """v0.0.20 改：调 orchestrator.cancel() 而非仅设 _stop flag。"""
        self._stop = True
        self._orchestrator.cancel()
```

```python
    """双层 token 预算注入工具清单。

    L1 工具清单：10 工具名 + 1 行 desc（按 permission_level 排序，read_only 优先）
    L2 工具详情：按剩余预算展开 desc + signature
    降级：超预算时只列工具名 + 权限级 + desc 截断到 30 字符
    """
    sorted_tools = sorted(
        available_tools,
        key=lambda t: {"read_only": 0, "write": 1, "destructive": 2}[t.permission_level],
    )
    # L1 + L2 合并
    lines = ["# 可用工具"]
    for t in sorted_tools:
        lines.append(f"- {t.name}（{t.permission_level}）：{t.description[:80]}")
    if len("\n".join(lines)) <= token_budget:
        return "\n".join(lines)
    # 降级：desc 截断到 30 字符
    lines = ["# 可用工具"] + [
        f"- {t.name}（{t.permission_level}）：{t.description[:30]}" for t in sorted_tools
    ]
    if len("\n".join(lines)) <= token_budget:
        return "\n".join(lines)
    # 最终降级：只列工具名 + 权限级
    return "\n".join(
        ["# 可用工具"] + [f"- {t.name}（{t.permission_level}）" for t in sorted_tools]
    )
```

`Orchestrator.run_turn` 第 1 步初始化时调 `planner.build_system_prompt(all_tools())` 拼到 system prompt。

```python
from q_agent.tools.registry import tool
```

```python
    subject: str
    description: str
    status: Literal["pending", "in_progress", "completed", "deleted"]

    name="todo_write",
    desc="更新当前会话任务列表（覆盖写）。"

    from pathlib import Path
    import json
    todo_dir = Path.home() / ".q-agent" / "todos"
    todo_dir.mkdir(parents=True, exist_ok=True)
    path = todo_dir / f"{session_id}.json"
```

**session_id 参数**：避免模块级变量多会话串号，由编排层调用时从 TurnState 传入。
