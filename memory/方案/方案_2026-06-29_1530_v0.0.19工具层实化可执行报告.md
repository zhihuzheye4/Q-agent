## 一、执行总览

### 1.1 v0.0.19 范围确认

v0.0.19 的核心目标是**把工具层从骨架实化为可被编排层调用的 10 个真实工具**，同时完成安全、审批、UI 反馈、持久化四条支撑线。范围按优先级分三档：

| 优先级 | 工具/能力 | 说明 |
|--------|----------|------|
| P0（必做） | `file_read` / `file_write` / `file_edit` / `file_list_dir` / `exec_shell` / `search_content` / `search_glob` | 读写+精确编辑+shell+检索，覆盖 90% 编码场景 |
| P1（必做） | `file_move` / `web_get` | 文件搬运 + 本地 Ollama 之外的轻量 HTTP |
| P2（必做） | `web_fetch` | 抓取并转 markdown，1MB 截断 |

**file_edit 是 2026-06-29 讨论后追加的 P0 工具**：三款主流框架（Claude Code / OpenHands / OpenClaw）都有独立 edit 工具，比 file_write 全覆盖更基础——精确改动只动该动的行，不破坏非改动区域，LLM 上下文也省（不用传整个文件）。原 9 工具方案补此 1 工具变 10 工具。

P0/P1/P2 全部在 v0.0.19 实装，**不留 P3 到 v0.0.20**——v0.0.20 只做编排层接通，不再加新工具。

支撑线（与工具同步落地）：
- 安全扩展：`safety.check_sensitive` + 内容嗅探 + URL 白名单 + SSRF 防护
- 审批机制：三级权限 + 弹窗 widget + 永不沙箱风险知情一次性弹窗
- UI 反馈：`ToolBubble` widget + 信号通道（v0.0.19 建好，v0.0.20 触发）
- 持久化：`tool_calls` 表 + `tool_audit` 表 + 落盘目录 `~/.q-agent/tool-results/`
- 撤销：写前快照备份 + git 兜底（Aider 模式）

## 三、工具注册与协议设计

### 3.1 `@tool` 装饰器最终签名

```python
# q_agent/tools/registry.py
from collections.abc import Callable
from typing import Any, Literal
import inspect

from q_agent.tools.base import ToolMeta
from q_agent.tools.spec import ToolSpec, PermissionLevel

_TOOLS: dict[str, ToolMeta] = {}


def tool(
    name: str,
    desc: str = "",
    *,
    version: str = "1.0.0",
    timeout: float | None = None,            # None = 走工具类默认
    long_running: bool = False,              # shell 类专用，True → 600s
    concurrency_safe: bool = False,          # v0.0.19 全 False，预留
    permission_level: PermissionLevel = "read_only",
    needs_confirmation: bool | None = None,  # None = 按 permission_level 推导
) -> Callable[[Callable[..., Any]], Callable[..., Any]]:
    """装饰器：登记工具函数 + 元数据 + 协议字段。

    参数说明：
    - name: snake_case，前缀 file_/search_/exec_/web_
    - desc: ≤200 字中文，四要素模板
    - version: 语义化 major.minor.patch，回放时宽松兼容
    - timeout: 秒；None 走工具类默认（read 15 / write 10 / shell 120 / http 30）
    - long_running: 仅 exec_shell，True 时 timeout 提升到 600
    - concurrency_safe: v0.0.19 串行，全 False；v0.0.21+ 多线程时再标 True
    - permission_level: read_only / write / destructive
    - needs_confirmation: None 时按 destructive→True / write→True / read_only→False 推导
    """
    def deco(fn: Callable[..., Any]) -> Callable[..., Any]:
        if name in _TOOLS:
            raise ValueError(f"工具名冲突: {name}")
        meta = ToolMeta(
            name=name,
            desc=desc,
            fn=fn,
            signature=str(inspect.signature(fn)),
            version=version,
            timeout=timeout,
            long_running=long_running,
            concurrency_safe=concurrency_safe,
            permission_level=permission_level,
            needs_confirmation=(
                needs_confirmation if needs_confirmation is not None
                else permission_level in ("write", "destructive")
            ),
        )
        _TOOLS[name] = meta
        return fn
    return deco


def lookup(name: str) -> ToolMeta | None:
    return _TOOLS.get(name)


def lookup_spec(name: str) -> ToolSpec | None:
    """返回 ToolSpec 视图（供编排层 / 外部源使用）。"""
    meta = _TOOLS.get(name)
    return meta.to_spec() if meta else None


def all_tools() -> list[ToolMeta]:
    return list(_TOOLS.values())
```

`ToolMeta` 扩展为 dataclass：

### 3.2 `ToolSpec` / `ExternalToolSource` Protocol 完整定义

```python
# q_agent/tools/spec.py
from typing import Protocol, Any, Literal, runtime_checkable
from collections.abc import Callable

PermissionLevel = Literal["read_only", "write", "destructive"]


class ToolSpec:
    """工具协议规范：内置 @tool 与外部 tool 源（MCP 等）统一形态。

    v0.0.19 内置工具通过 ToolMeta.to_spec() 转换；
    v0.0.21+ MCP 外部源直接实现 ExternalToolSource Protocol 返回 ToolSpec 列表。
    """
    name: str
    description: str
    version: str
    input_schema: str          # inspect.signature 字符串或 JSON Schema
    output_schema: str         # "str" 或 JSON Schema
    concurrency_safe: bool
    permission_level: PermissionLevel
    needs_confirmation: bool
    timeout: float | None
    call: Callable[..., str]   # 实际调用入口

    def __init__(
        self,
        name: str,
        description: str,
        version: str,
        input_schema: str,
        output_schema: str,
        concurrency_safe: bool,
        permission_level: PermissionLevel,
        needs_confirmation: bool,
        timeout: float | None,
        call: Callable[..., str],
    ) -> None:
        self.name = name
        self.description = description
        self.version = version
        self.input_schema = input_schema
        self.output_schema = output_schema
        self.concurrency_safe = concurrency_safe
        self.permission_level = permission_level
        self.needs_confirmation = needs_confirmation
        self.timeout = timeout
        self.call = call


@runtime_checkable
class ExternalToolSource(Protocol):
    """外部工具源协议：v0.0.21+ MCP 适配器实现此 Protocol 即可零侵入接入。

    编排层只认 ToolSpec，不认具体源。
    """
    def list_tools(self) -> list[ToolSpec]: ...
    def get_tool(self, name: str) -> ToolSpec | None: ...
    def refresh(self) -> None: ...
```

## 四、10 个工具的实现规范

### 4.1 `file_read`

```python
# q_agent/tools/files.py
from q_agent.tools.registry import tool
from q_agent.tools.safety import check_path, check_sensitive

@tool(
    name="file_read",
    desc=(
        "读取文本文件内容并返回字符串。"
        "何时用：需要查看源代码、配置文件、文档内容时。"
        "何时不用：读取二进制文件（图片/音频/可执行）、读取超过 1MB 的大文件、读取敏感凭证文件。"
        "参数约束：path 必须为绝对路径或相对 cwd 的路径；命中敏感清单返回 PermissionError。"
        "返回格式：文件全文（超 2000 字符触发预算降级）。"
    ),
    version="1.0.0",
    timeout=15.0,
    permission_level="read_only",
)
def file_read(path: str, encoding: str = "utf-8") -> str:
    check_path(path)
    check_sensitive(path)  # 命中 .env/.pem/id_rsa/credentials.json 等拦截
    from pathlib import Path
    p = Path(path)
    if not p.exists():
        return _error_json("FileNotFound", f"文件不存在: {path}")
    if p.stat().st_size > 1_000_000:
        return _error_json("FileTooLarge", f"文件超过 1MB: {p.stat().st_size} bytes")
    return p.read_text(encoding=encoding)
```

**错误回喂 JSON 格式**（所有工具统一）：

```json
{"error": "FileNotFound", "message": "文件不存在: /foo/bar.py", "recoverable": true}
```

`recoverable: true` 表示 LLM 可自愈重试（改路径 / 改参数），`recoverable: false` 表示 PermissionError 类硬拦截（零重试）。

**撤销策略**：file_read 不适用撤销（只读）。

### 4.2 `file_write`

```python
@tool(
    name="file_write",
    desc=(
        "将字符串内容写入文件（覆盖模式）。"
        "何时用：创建新文件、修改源代码、生成配置文件。"
        "何时不用：追加内容（用 file_append，v0.0.20 再加）、写二进制。"
        "参数约束：path + content；命中敏感文件或内容含 PRIVATE KEY/AKIA 模式拒绝。"
        "返回格式：写入字节数 + 落盘备份路径。"
    ),
    version="1.0.0",
    timeout=10.0,
    permission_level="write",
)
def file_write(path: str, content: str, encoding: str = "utf-8") -> str:
    check_path(path)
    check_sensitive(path)
    _sniff_secret_content(content)  # 命中 PRIVATE KEY/AKIA[0-9A-Z]{16} 等拒绝
    from pathlib import Path
    from q_agent.tools.undo import snapshot_before_write
    backup = snapshot_before_write(path)  # 写前快照，存在则备份到 ~/.q-agent/snapshots/
    Path(path).write_text(content, encoding=encoding)
    return f"写入 {len(content.encode(encoding))} bytes; backup={backup}"
```

**撤销策略**：写前快照备份到 `~/.q-agent/snapshots/{call_id}/{原文件名}`，若文件原不存在则不备份但记录"新建"标记；同时维护 undo 栈（最近 20 条），LLM 调 `undo_last_write`（v0.0.20 编排层暴露）回滚。

### 4.3 `file_edit`

```python
@tool(
    name="file_edit",
    desc=(
        "在既有文件中做精确字符串替换（首次匹配替换）。"
        "何时用：修改源代码中某个函数 / 改配置文件某行 / 重命名变量。"
        "何时不用：全覆盖写（用 file_write）、追加内容（v0.0.20 再加 file_append）、批量同字符串替换（用 search_content 先定位再多次调）。"
        "参数约束：path + old_string + new_string；old_string 必须在文件中唯一出现，否则返回 AmbiguousMatch 错误（recoverable: true，要求 LLM 提供更多上下文行）；命中敏感文件或内容含 PRIVATE KEY/AKIA 模式拒绝。"
        "返回格式：替换成功字节数 + 落盘快照路径 + 命中行号。"
    ),
    version="1.0.0",
    timeout=10.0,
    permission_level="write",
)
def file_edit(path: str, old_string: str, new_string: str, encoding: str = "utf-8") -> str:
    if old_string == "":
        return _error_json("EmptyOldString", "old_string 不能为空", recoverable=True)
    if old_string == new_string:
        return _error_json("NoOp", "old_string 与 new_string 相同，无需替换", recoverable=True)
    check_path(path)
    check_sensitive(path)
    _sniff_secret_content(new_string)
    from pathlib import Path
    from q_agent.tools.undo import snapshot_before_write
    p = Path(path)
    if not p.exists():
        return _error_json("FileNotFound", f"文件不存在: {path}", recoverable=True)
    content = p.read_text(encoding=encoding)
    occurrences = content.count(old_string)
    if occurrences == 0:
        return _error_json("StringNotFound", "old_string 未在文件中找到", recoverable=True)
    if occurrences > 1:
        return _error_json("AmbiguousMatch", f"old_string 在文件中出现 {occurrences} 次，需提供更多上下文行", recoverable=True)
    backup = snapshot_before_write(path)
    new_content = content.replace(old_string, new_string, 1)
    p.write_text(new_content, encoding=encoding)
    hit_line = content[: content.index(old_string)].count("\n") + 1
    return f"替换 1 处; hit_line={hit_line}; backup={backup}"
```

**撤销策略**：与 file_write 相同——写前快照备份到 `~/.q-agent/snapshots/{call_id}/{原文件名}`，undo 栈记录（最近 20 条）。

**与 file_write 的关系**：
- `file_write` = 全覆盖写（创建新文件 / 重写整个文件）
- `file_edit` = 精确替换（修改既有文件中的某段）
- LLM 应优先用 `file_edit` 改既有文件（上下文省、不破坏非改动区域），仅在创建新文件或大重构时用 `file_write`
- 两者权限级都是 `write`（首次需确认，args_hash 缓存）

```python
@tool(
    name="search_content",
    desc=(
        "在文件内容中搜索正则或字面量（递归）。"
        "何时用：查找某函数定义、某变量引用、某字符串出现位置。"
        "何时不用：按文件名查找（用 search_glob）、单文件内搜索（用 file_read 后自处理）。"
        "参数约束：root 必须为目录；pattern 为正则；max_results 默认 50。"
        "返回格式：每行 path:line:matched_text。"
    ),
    version="1.0.0",
    timeout=15.0,
    permission_level="read_only",
)
def search_content(root: str, pattern: str, max_results: int = 50) -> str:
    check_path(root)
    import re
    from pathlib import Path
    rx = re.compile(pattern)
    hits = []
    for p in Path(root).rglob("*"):
        if not p.is_file() or p.stat().st_size > 500_000:
            continue
        try:
            for i, line in enumerate(p.read_text(encoding="utf-8", errors="ignore").splitlines(), 1):
                if rx.search(line):
                    hits.append(f"{p}:{i}:{line[:120]}")
                    if len(hits) >= max_results:
                        return "\n".join(hits) + f"\n[截断：达 max_results={max_results}]"
        except (PermissionError, OSError):
            continue
    return "\n".join(hits) if hits else "无匹配"
```

```python
# q_agent/tools/safety.py（在既有 check_command/check_path 基础上追加）

from pathlib import Path
import re
import ipaddress
import urllib.parse

# 敏感文件清单（命中即拒绝 read/write）
SENSITIVE_PATTERNS: tuple[str, ...] = (
    ".env", ".env.*", "*.pem", "*.key", "id_rsa", "id_rsa.*",
    "credentials.json", "credentials", ".aws/credentials",
    ".git-credentials", "*.keystore", "*.p12",
)

# 用户自定义敏感文件 glob（运行时加载）
SENSITIVE_USER_FILE = Path.home() / ".q-agent" / "sensitive.txt"


def check_sensitive(path: str) -> None:
    """命中敏感文件清单 → PermissionError（不回喂 recoverable，零重试）。"""
    p = Path(path).resolve()
    name = p.name
    parent_str = str(p)
    # 内置清单匹配
    for pat in SENSITIVE_PATTERNS:
        if _glob_match(pat, name) or pat in parent_str:
            raise PermissionError(f"敏感文件被拦截: {path}")
    # 用户自定义清单
    if SENSITIVE_USER_FILE.exists():
        for line in SENSITIVE_USER_FILE.read_text(encoding="utf-8").splitlines():
            line = line.strip()
            if line and not line.startswith("#") and _glob_match(line, name):
                raise PermissionError(f"用户自定义敏感文件被拦截: {path}")


SECRET_CONTENT_PATTERNS = (
    re.compile(r"-----BEGIN (RSA |EC |OPENSSH |)PRIVATE KEY-----"),
    re.compile(r"AKIA[0-9A-Z]{16}"),          # AWS access key
    re.compile(r"ghp_[0-9A-Za-z]{36}"),       # GitHub PAT
    re.compile(r"sk-[0-9A-Za-z]{20,}"),       # OpenAI key 风格
)


def sniff_secret_content(content: str) -> None:
    """写入内容嗅探：命中私密凭证模式 → PermissionError。"""
    for rx in SECRET_CONTENT_PATTERNS:
        if rx.search(content):
            raise PermissionError("写入内容含私密凭证模式，拒绝")


URL_ALLOWLIST: set[str] = {
    "pypi.org", "github.com", "raw.githubusercontent.com",
    "huggingface.co", "docs.python.org", "localhost:11434",
}

PRIVATE_RANGES = (
    ipaddress.ip_network("10.0.0.0/8"),
    ipaddress.ip_network("172.16.0.0/12"),
    ipaddress.ip_network("192.168.0.0/16"),
    ipaddress.ip_network("127.0.0.0/8"),
    ipaddress.ip_network("169.254.0.0/16"),
    ipaddress.ip_network("::1/128"),
    ipaddress.ip_network("fc00::/7"),
)


def check_url(url: str) -> None:
    """强制 HTTPS + 域名白名单。"""
    parsed = urllib.parse.urlparse(url)
    if parsed.scheme != "https" and parsed.netloc != "localhost:11434":
        raise PermissionError(f"仅允许 HTTPS: {url}")
    host = parsed.hostname or ""
    allowed = any(host == h or host.endswith("." + h.split(":")[0]) for h in URL_ALLOWLIST)
    if not allowed:
        raise PermissionError(f"域名不在白名单: {host}")


def check_ssrf(url: str) -> None:
    """私网段拒绝（SSRF 防护）。"""
    parsed = urllib.parse.urlparse(url)
    host = parsed.hostname or ""
    try:
        ip = ipaddress.ip_address(host)
    except ValueError:
        return  # 域名走 DNS，由 urllib 自行解析；v0.0.19 不做 DNS rebinding 防护
    for net in PRIVATE_RANGES:
        if ip in net:
            raise PermissionError(f"私网段被拦截: {ip}")
```

### 5.3 永不沙箱风险知情流程

```python
# q_agent/tools/approval.py
import hashlib, json
from pathlib import Path

SETTINGS_FILE = Path.home() / ".q-agent" / "settings.json"


def needs_sandbox_risk_ack() -> bool:
    """首次使用 shell 前检查 settings 是否已确认。"""
    if not SETTINGS_FILE.exists():
        return True
    data = json.loads(SETTINGS_FILE.read_text(encoding="utf-8"))
    return not data.get("sandbox_risk_acked", False)


def ack_sandbox_risk() -> None:
    SETTINGS_FILE.parent.mkdir(parents=True, exist_ok=True)
    data = {}
    if SETTINGS_FILE.exists():
        data = json.loads(SETTINGS_FILE.read_text(encoding="utf-8"))
    data["sandbox_risk_acked"] = True
    SETTINGS_FILE.write_text(json.dumps(data, ensure_ascii=False, indent=2), encoding="utf-8")


def args_hash(tool_name: str, args: tuple, kwargs: dict) -> str:
    """同会话缓存键：tool_name + 参数指纹。"""
    payload = json.dumps({"t": tool_name, "a": list(args), "k": kwargs}, sort_keys=True, default=str)
    return hashlib.sha256(payload.encode("utf-8")).hexdigest()

        rows = load_tool_calls()
        self.table.setRowCount(len(rows))
        for r, row in enumerate(rows):
            for c, val in enumerate(row):
                self.table.setItem(r, c, QTableWidgetItem(str(val)))
```

### 7.3 ContextManager L1 与 tool_calls 表的联动

```python
# q_agent/orchestrator/context.py（L1 占位降级逻辑预览，v0.0.20 接通）
# 当会话总预算 24000 字符超限时，按三档分级降级：
# L1 占位：tool_call 输出替换为 "[tool:{tool_name} output 已落盘 {path}]"
# L2 截断：保留前 200 + 后 200 字符，中间 ...截断...
# L3 摘要：调 LLM 生成 100 字摘要替换

def compact_tool_outputs(self, session_id: str) -> None:
    """会话总预算超限时，按 L1→L2→L3 逐档降级 tool_calls 输出。"""
    rows = self.persistence.load_tool_calls(session_id)
    total = sum(len(r["output_text"] or "") for r in rows)
    if total <= 24000:
        return
    # 先 L1 占位（最早 N 条）
    for r in rows:
        if total <= 24000:
            break
        total -= len(r["output_text"] or "")
        new_text = f"[tool:{r['tool_name']} output 已落盘 {r['output_path']}]"
        self.persistence.update_tool_call_output(r["call_id"], new_text)
        total += len(new_text)
```

# v0.0.20 dispatcher.py 预览（不在 v0.0.19 实施）
from q_agent.tools.registry import lookup_spec
from q_agent.tools.executor import execute_tool_with_lifecycle

def handle_tool_call(self, name: str, args: dict) -> str:
    spec = lookup_spec(name)
    if spec is None:
        return _error_json("ToolNotFound", name, recoverable=False)

**判定依据**：
- 这些功能影响**全局状态**（会话切换 / plan 模式 / 子 agent 派发），不应让 LLM 自由调用——由框架控制更可控
- 本地小模型（qwen2.5 等）显式调用这类"元工具"容易出错（上下文不足时误判该不该 spawn 子 agent），框架隐式管理对小模型更友好
- 与三款框架的设计哲学差异：Claude Code 偏向"LLM 自治"，Q-agent 偏向"框架可控"
