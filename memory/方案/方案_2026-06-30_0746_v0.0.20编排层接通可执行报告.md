## 二、Claude Code 调研结论 + Q-agent 适配决策

### 2.1 Plan 模式

**Claude Code 行为**：LLM 自主调 `EnterPlanMode` 进入研究态 → 委托独立只读 Plan 子 agent（跳过 CLAUDE.md + git status 提速）→ ExitPlanMode 需用户授权 → plan 文本是工具调用参数形态，不强制落盘 → 用户否决后 LLM 留原会话调整重提。

**Q-agent 适配**：**不派发独立 Plan 子 agent**——本地小模型不存在 Haiku 级快速模型，派发独立子 agent 反而拖慢。改为**父 agent 内同会话切权限态**：
- `EnterPlanMode` 工具调用后，TurnState 加 `plan_mode: bool` 字段
- `plan_mode=True` 时，dispatcher 拒绝所有 write/destructive 工具调用（直接回喂 PermissionError "plan 模式禁写"）
- `ExitPlanMode` 工具调用携带 plan 文本 → UI 弹窗审批 → 用户确认后 plan 文本落盘到 `~/.q-agent/plans/{session_id}/{ts}.md` + 退出 plan 模式
- 用户否决 plan → 留 plan 模式 + 把否决理由回喂 LLM 继续研究

### 2.2 子 Agent 派发（v0.0.20 不实施，挪到后续单独里程碑）

**Claude Code 行为**：LLM 显式调 `Agent` 工具（v2.1.63 由 Task 改名）→ 子 agent 独立干净上下文（不继承父对话）→ 父只收子最终文本（不见中间工具调用）→ 嵌套 5 层 / fork / worktree 隔离。

**Q-agent 适配方向（仅留档，v0.0.20 不实施）**：2026-06-30 用户明确指示"子 agent 暂时不考虑，后面会有单独开发环节"。v0.0.20 范围不含 Agent 子 agent 派发，本节作为后续单独里程碑的调研留档保留。适配思路：**砍掉嵌套 5 层/fork/worktree**——保留单层、独立上下文、最终文本回喂即可（详见后续单独里程碑方案）。

### 2.3 TodoWrite 任务管理

**Claude Code 行为**：v2.1.142 起 TodoWrite 默认禁用，改 TaskCreate/TaskGet/TaskList/TaskUpdate 四件套（CLAUDE_CODE_TASK_LIST_ID 跨会话共享）→ JSONL 落盘 → Ctrl+T 独立浮层 UI（最多 5 条）→ 规划阶段一次性写入 + 执行中动态更新。

**Q-agent 适配**：**单一 TodoWrite 工具 + 会话内 JSON 持久化**——小模型调四件套过复杂：
- `TodoWrite` 工具签名：`todo_write(tasks: list[dict]) -> str`，每项 `{subject, description, status, activeForm}`
- status 四态：pending / in_progress / completed / deleted
- 持久化：会话内 JSON 落盘到 `~/.q-agent/todos/{session_id}.json`，切换会话保留（同会话内跨压缩保留，跨会话不共享）
- UI 展示：技能 tab 下方新增"任务"区域，与已注册技能/已开发工具并列展示（不独立浮层，避免新增 widget）
- LLM 调用时机：规划阶段一次性写入 + 执行中动态更新（与 Claude Code 对齐）

### 2.4 Sessions_* 会话管理

**Claude Code 行为**：sessions 是用户命令（`/resume`、`/clear`、`/compact`、`--resume`）→ JSONL 落盘 `~/.claude/projects/{project}/{sessionId}/` → 切换是"保留+恢复"模型 → 子 agent 即新会话单元（嵌套最深 5 层）。

**Q-agent 适配**：**作为 LLM 可调工具 + sqlite 持久化**——Q-agent 哲学是"让 LLM 指令被软件理解执行"，sessions 锁死在用户命令层违背此哲学：
- 3 个工具：`sessions_list()` / `sessions_switch(session_id)` / `sessions_new()`
- 持久化：复用 v0.0.18 `persistence.py` 的 sessions 表（已建）
- 切换上下文：清空当前内存消息历史 + `load_messages(limit=N)` 加载新会话历史（与 Claude Code "保留+恢复"模型对齐——不删旧会话，可后续 switch 回来）
- UI：会话切换由 LLM 工具调用触发，UI 自动刷新消息流；未来 v0.0.21+ 可加用户命令 `/sessions` 列表选择

### 3.2 M2 system prompt 工具段注入

```python
# q_agent/orchestrator/planner.py（扩展）
def build_system_prompt(
    self,
    available_tools: list[ToolSpec],
    token_budget: int = 2000,
) -> str:
    """双层 token 预算注入工具清单。

    L1 工具清单：10 工具名 + 1 行 desc（按 permission_level 排序，read_only 优先）
    L2 工具详情：按剩余预算展开 desc + signature
    """
    # 按 read_only → write → destructive 排序
    sorted_tools = sorted(
        available_tools,
        key=lambda t: {"read_only": 0, "write": 1, "destructive": 2}[t.permission_level],
    )
    lines = ["# 可用工具"]
    for t in sorted_tools:
        lines.append(f"- {t.name}（{t.permission_level}）：{t.description[:80]}")
    if len("\n".join(lines)) > token_budget:
        # 降级：只列工具名 + 权限级，desc 截断到 30 字符
        lines = ["# 可用工具"] + [f"- {t.name}（{t.permission_level}）" for t in sorted_tools]
    return "\n".join(lines)
```

`Orchestrator.run_turn` 第 1 步初始化时调 `planner.build_system_prompt(all_tools())` 拼到 system prompt。

def todo_write(tasks: list[dict]) -> str:

    import json
    todo_dir = Path.home() / ".q-agent" / "todos"
